centos7下安装python3.6.1

发表于 2018-04-29 | 分类于 linux ， operations
字数统计: 1,130

不要动现有的python2环境！因为有很多系统命令，比如yum都要用到。

1. 安装依赖环境

部分依赖包说明:
gcc:linux ( centos ) gcc 编译环境 ,安装后可以编译 Python 源码 –> make &; make install
zlib-devel:压缩依赖包
openssl-devel:让 Python pip 可以链接 https
bzip2-devel:让 linux( centos) 可以使用bzip2 压缩
sqlite-devel:sqlite 数据库依赖包 , 让 Python 可以使用 sqlite 数据库 –> import sqlite3

2.下载Python3

wget https://www.python.org/ftp/python/3.6.1/Python-3.6.1.tgz

3.安装python3

安装在/usr/local/python3（具体安装位置看个人喜好）
创建目录：
mkdir -p /usr/local/python3
tar -zxvf Python-3.6.1.tgz

4.进入解压后的目录，编译安装。

cd Python-3.6.1
./configure –prefix=/usr/local/python3
{
./configure命令执行完毕之后创建一个文件creating Makefile，供下面的make命令使用 执行make install之后就会把程序安装到我们指定的目录中去。
在待安装的源码路径下使用命令./configure –help输出详细的选项列表。
其中–prefix选项是配置安装的路径，如果不配置该选项，安装后可执行文件默认放在/usr /local/bin，库文件默认放
在/usr/local/lib，配置文件默认放在/usr/local/etc，其它的资源文件放在/usr /local/share，比较凌乱。
如果配置–prefix，如：
./configure –prefix=/usr/local/test
可以把所有资源文件放在/usr/local/test的路径中，不会杂乱。
用了—prefix选项的另一个好处是卸载软件或移植软件。当某个安装的软件不再需要时，只须简单的删除该安装目录，
就可以把软件卸载得干干净净；移植软件只需拷贝整个目录到另外一个机器即可（相同的操作系统）。
当然要卸载程序，也可以在原来的make目录下用一次make uninstall，但前提是make文件指定过uninstall。
}

make && make install

5.修改 Python 环境变量 或软连接修改环境变量

软连接修改环境变量实例:建立python3的软链,使其在终端输入 Python ,直接跳转到 Python3
● A、重命名 /usr/bin/python为 python2.7,防止和之前软链接冲突
● B、创建软链接到 编译好的 Python3 目录

mv /usr/bin/python /usr/bin/python2.7
执行ln -s /usr/local/python3/bin/python3 /usr/bin/python命令创建软连接。执行python -V查看Python是否安装成功。
ln -s /usr/local/python3/bin/python3 /usr/bin/python3
ln -s /usr/local/python3/bin/pip3 /usr/bin/pip #pip是Python官方推荐的包管理工具。

6.修改yum等相关配置文件

因为yum使用python2，因此替换为python3后可能无法正常工作，继续使用这个python2.7
因此修改yum配置文件(vi /usr/bin/yum)。
把文件头部的#!/usr/bin/python改成#!/usr/bin/python2.7保存退出即可。

vi /usr/libexec/urlgrabber-ext-down
把文件头部的#!/usr/bin/python改成#!/usr/bin/python2.7保存退出即可。
{
update 2018/07/08 11:26 centos7下执行firewall-cmd显示ImportError: No module named ‘gi’
出现这个问题是，更新了Python版本（version>=Python3），cnetos7系统自带的是Python2.7。解决办法:

  1. yum install python3-gi

或者2. 修改相关配置文件：
第一步，vim /usr/bin/firewall-cmd， 将#！/usr/bin/python -Es 改为 #！/usr/bin/python2.7 -Es（到目前为止，上面提到的问题已解决）
顺便把这两个文件一并改了:
/usr/bin/firewall-config
/usr/bin/firewall-offline-cmd

第二步，vim /usr/sbin/firewalld, 将#！/usr/bin/python -Es 改为 #！/usr/bin/python2.7 -Es (这一步是针对于防火墙起不来的问题(4月底安装python3之后就有问题,不过一直没发现)，进行的修改—>如何查询CentOS7开关机日志中的报错??)
改完后重启firewalld,ok!

发现别人也遇到过这个问题:Centos7 安装python3后不兼容问题

除此之外还有一些其他文件使用的Python，可以使用grep命令查看哪些文件使用了Python解释器，根据具体情况修改。例如查看/usr/bin目录下的文件
grep -ai /usr/bin/python /usr/bin/* | wc -l 统计有66个,令人头秃….
}

7、设置别名方便使用

alias py=python3

8、升级安装最新版PIP

pip install --upgrade pip
